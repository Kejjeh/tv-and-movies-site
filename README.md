# Static UI

Vanilla HTML / CSS / JS. No build step. Reads `data.json` (generated by
`scripts/export_to_json.py`) and runs the 4-dimension scoring matrix
client-side in JavaScript — a direct port of `brain/scoring.py`.

## Local preview

```
python scripts/export_to_json.py     # rebuild data.json
cd web
python -m http.server 8765
# open http://localhost:8765
```

(Opening `index.html` via `file://` won't work because `fetch` blocks
local file URLs in most browsers.)

## Files

- `index.html` — page layout
- `styles.css` — dark theme
- `app.js` — scoring port + rendering
- `data.json` — exported title/people/scoring-config payload

## Deployment

GitHub Action `.github/workflows/deploy-site.yml` rebuilds the JSON
on every push that touches the YAML, the SPA, or scoring, and pushes
the `web/` directory to `Kejjeh/tv-and-movies-site`, where GitHub
Pages serves it. See `SETUP_SITE.md` at the repo root for the one-time
manual setup steps (creating the public repo, configuring PAT and
API key secrets).
